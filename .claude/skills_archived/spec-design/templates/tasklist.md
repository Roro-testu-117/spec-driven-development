# タスクリスト

## ノート
- `*` が付いたタスク（例: `[ ]* 3.2 ...`）は任意タスク。MVP短縮時のみスキップ可。
- 本テンプレートは小規模機能向け。重い進行管理（Checkpoint等）は置かない。

## タスク完了ルール
- 全てのタスクを `[x]` にすること
- 大項目は `N.`、小項目は `N.M` の番号を付けること
- 小項目ごとに「成果物（ファイル）」と「Done条件」を必ず記載する
- 1小項目に複数責務が混在する場合は分割する
- 1小項目の成果物（ファイル）が多すぎる場合（目安: 5件超）は分割する
- Done条件は観測可能な完了状態で記載する
- スキップ時は理由を明記する

## タスク記述テンプレート（必須）
- [ ] N. 大項目名
  - 目的
    - {この大項目で達成すること}
- [ ] N.M 小項目名
  - Depends on（任意）
    - {例: 2.1}
  - 成果物（ファイル）
    - [ ] path/to/file1
  - Done条件
    - [ ] 観測可能な完了状態

## フェーズ4: 品質チェック
- [ ] 4. リント・フォーマット・テストを実施する

    - コード品質を担保し、CI/CDで問題が起きない状態にする
- [ ] 4.1 リントチェック
  - 成果物（ファイル）

  - Done条件

- [ ] 4.2 フォーマットチェック

- [ ] 4.3 テスト実行
