# コードレビュー評価軸

## 評価項目

| # | 観点 | 評価基準 | 重要度 |
|---|------|---------|--------|
| 1 | 設計適合性 | design.md の実装方針に沿っているか | 高 |
| 2 | タスク完了度 | tasklist.md の全タスクが完了しているか | 高 |
| 3 | コード品質 | 命名規則、可読性、エラーハンドリング | 中 |
| 4 | セキュリティ | APIキー漏洩、機密情報の取り扱い | 高 |
| 5 | テスト | 必要なテストが書かれ、パスしているか | 中 |

## 評価記号

- ✓: 問題なし
- △: 軽微な指摘あり（修正推奨）
- ✗: 要修正

## 承認基準

- 全項目が ✓ または △: OK
- 1つでも ✗ がある: 修正後に再レビュー
- セキュリティが ✗ の場合: 即座に修正必須
