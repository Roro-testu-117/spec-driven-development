# LLM実装・レビュー規約

## 実装規約

1. 設定管理
- APIキーや機密値は環境変数経由で読み込む
- 環境依存値をハードコードしない

2. プロンプト管理
- 文字列直書きよりテンプレート化を優先する
- 入出力スキーマ（期待形式）を明示する

3. モデル呼び出し
- タイムアウトとリトライ方針を明示する
- モデル選定理由（精度/速度/コスト）を残す

4. RAG構成
- 検索戦略（k値、フィルタ、再ランキング）を明示する
- 参照元情報（citation）を可能な限り保持する

## テスト規約

- 単体テスト: プロンプト整形、パーサ、分岐ロジックを優先
- 結合テスト: Retriever -> LLM -> 出力整形の主要経路を確認
- 回帰テスト: 重要プロンプトの期待出力をスナップショット等で管理
- 失敗系テスト: API失敗・空検索結果・タイムアウトを確認

## レビュー規約

- 設計との整合性（flow, interface, error handling）を優先確認
- スコープ外変更がないかを確認
- 秘密情報漏えい、危険なログ出力、未検証入力を重点チェック
- 性能劣化が疑われる変更は、少なくとも1つ定量指標で確認
