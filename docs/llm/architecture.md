# LLMアーキテクチャ原則

## 対象

- LangChain 1.0+
- LangGraph
- OpenAI / Anthropic / Google Generative AI
- ベクトルDB（例: Chroma）

## 原則

1. コンポーネント分離
- Prompt構築、Retriever、Model呼び出し、Post-process を分離する
- 役割ごとにモジュールを分け、再利用しやすくする

2. データフローの明確化
- 入力検証 -> 取得 -> 生成 -> 出力整形 の流れを固定する
- 中間データ（context, citations, metadata）を追跡可能にする

3. 失敗時挙動の定義
- LLM/API失敗時のリトライ、タイムアウト、フォールバックを明示する
- エラーは握りつぶさず、利用者に意味のある形で返す

4. 観測可能性
- トレースID、レイテンシ、トークン使用量、失敗率を収集する
- 重大な失敗を検知できるログ粒度を維持する

5. 安全性
- 秘密情報はコードへ埋め込まない
- プロンプト注入や不正入力を前提に境界チェックを行う

## LangGraph 利用時の補足

- ノード責務を単一化し、状態遷移を明示する
- 再実行可能性を考慮し、副作用のある処理を限定する
- ヒューマンインザループが必要な判定点を明確にする
